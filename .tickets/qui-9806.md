# Responsive UI improvements

Improve the application experience across desktop, tablet, and mobile. The app should remain usable on narrow viewports, with accessible navigation, controls, Kanban interactions, and graph interactions.

## Design

Keep this separate from epic filtering because responsive UX cuts across the whole application. Use an audit first, then small focused fixes. New epic-filter UI should be designed with this broader responsive work in mind.

## Acceptance Criteria

Core views are usable on desktop, tablet, and mobile-sized viewports. Navigation and controls are not clipped or unreachable. Kanban and graph views have acceptable touch/scroll/control behavior. Responsive issues are addressed in small reviewable tickets. bun run sensors all passes.
